# Brand & UI Constitution — Human-First Edition

**Platform:** Bassan  
**Scope:** Platform Admin Panel + Client UI  
**Status:** Binding Direction (Docs-Only)  
**Philosophy:** Human-First, Calm Power, Elegant Speed  

---

## 1. Core Vision

Bassan is an operating system used for long hours.  
The UI must **reduce cognitive load**, not add to it.

The system should feel:
- Calm
- Confident
- Premium
- Supportive
- Fast without anxiety

> The UI exists to serve people, not to impress them.

---

## 2. Brand Scope & Application

- **One unified brand** across:
  - Platform Admin
  - Client (Tenant) UI
- Brand **intensity varies by context**:
  - Admin → denser, more operational
  - Client → calmer, lighter, more spacious

No brand fragmentation.  
No white-label feeling.

---

## 3. Visual Identity Principles

### Color
- Light theme is the default
- Dark theme is secondary (focus / preference)
- Primary color: **Muted Purple**
  - Neutral
  - Non-aggressive
  - Supports long usage

### Contrast
- Soft backgrounds
- Controlled contrast
- No harsh color jumps

---

## 4. Logo Usage

- Logo is **always present**
- Small
- Calm
- Never dominant

Logo exists to reassure, not to advertise.

---

## 5. Motion & Animation

Motion is **functional**, not decorative.

Rules:
- Duration: 150–200ms
- No bounce
- No exaggerated easing
- No sound
- Motion guides orientation only

> Motion should feel like breathing — not talking.

---

## 6. Navigation System

### Sidebar Behavior
- Sidebar may:
  - Slide in/out
  - Act as overlay (drawer)
- Sidebar must feel:
  - Attached to the surface
  - Not floating above it

### Active State
- Active navigation item:
  - Inherits page color (softly)
  - Indicates context without text

### Safety Rules
- During critical operations:
  - Sidebar remains stable
  - No animated disappearance

---

## 7. Page Structure Strategy

- Fewer pages than features
- Prefer:
  - Sections
  - Drawers
  - Tabs
- Avoid excessive page hopping

Icons are **shortcuts**, not replacements.

---

## 8. Advanced Operations

Heavy or risky operations must:
- Be explicit
- Be separated visually
- Enter a calmer, more focused mode

Example:
- “Advanced”
- “Manage”
- “Full View”

> Advanced work deserves a different mental space.

---

## 9. Loading & Startup Experience

### System Opening
- Startup screen uses animated Bassan logo
- Purpose:
  - Warm-up
  - Data hydration
  - Orientation

### Loading Rules
- Prefer skeletons over spinners
- No blank screens
- Always show progress intent

---

## 10. Error & Feedback Philosophy

### Errors
- Clear
- Human
- Actionable
- Never blaming

### Success
- Minimal feedback
- No unnecessary confirmations

Silence means success.

---

## 11. Cognitive Safety Rules

- No surprise behavior
- No sudden layout changes
- No forced flows
- Undo when possible
- Back preserves state

The user must always feel **in control**.

---

## 12. Notification Discipline

- No noisy notifications
- No repeated alerts
- Important only

The system must never nag.

---

## 13. Typography & Language

- Human tone
- Simple sentences
- No technical jargon unless necessary
- Friendly but professional

The system speaks like a calm expert.

---

## 14. Emotional Design

The UI should make users feel:
- Confident
- Comfortable
- Respected
- Supported

Never rushed.  
Never judged.

---

## 15. Performance Perception

Speed is not only performance.
It is **perceived responsiveness**.

Rules:
- Immediate visual feedback
- No dead clicks
- No uncertain states

---

## 16. Design Governance Rules

Any new UI element must answer:
1. Does it reduce mental effort?
2. Does it repeat often?
3. Does it justify its motion?

If not → it does not belong.

---

## 17. Future Protection Rules

- No feature breaks established motion language
- No new pattern without purpose
- No visual noise creep

Consistency is a feature.

---

## 18. Client Experience Safeguards

- Client UI:
  - Lighter density
  - More whitespace
  - Fewer controls visible by default
- Advanced features remain accessible but hidden

---

## 19. Brand as a Silent Advantage

Bassan should feel premium **without trying**.

No showing off.  
No over-design.  
No unnecessary complexity.

---

## 20. Final Principle

> If the user feels calm after 8 hours of work,  
> the UI has succeeded.

This constitution is binding and evolves only through governed design gates.
