# Commit & PR Rules

---

## Commit Requirements

Each commit must:

- Have clear purpose
- Modify a single concern
- Avoid mixed responsibilities

Commit message format:

area: action description

Example:
platform-admin: fix organization fail-closed logic

---

## PR Requirements

PR must include:

- What changed
- Why it changed
- Verification steps
- Risks or rollback notes

---

## Forbidden Commits

Avoid:

- Mega commits
- Formatting-only commits
- Mixed feature and fix commits
